# CRUD Приложение - Управление пользователями

Тестовое задание на Nuxt 4 с использованием Pinia, Vuetify 3 и Tailwind CSS.
Проект реализован с применением архитектуры **Feature-Sliced Design (FSD)**.

## Технологический стек

- **Nuxt 4** - фреймворк
- **Pinia** - управление состоянием
- **Vuetify 3** - UI компоненты
- **Tailwind CSS** - для классов
- **TypeScript** - строгая типизация
- **FSD** - Feature-Sliced Design архитектура

## Реализованный функционал

###  Список пользователей (/)

- Таблица `<v-data-table>` (вынесена в UsersTable)
- Пагинация (10, 25, 50, 100 элементов)
- Поиск по всем полям
- Сортировка по ФИО и дате рождения
- Кнопки редактирования и удаления

###  Создание пользователя (/create)

- Форма с валидацией (переиспользуемый UserForm widget)
- Отображение ошибок
- Уведомления через AppSnackbar
- Сохранение в Pinia store

###  Редактирование пользователя

- Модальное окно EditUserDialog (feature)
- Предзаполнение формы
- Валидация через shared/lib
- Обновление в store

###  Удаление пользователя

- Диалог DeleteUserDialog (feature)
- Подтверждение действия
- Отображение информации
- Удаление из store

## Конфигурации 

Все конфигурации вынесены в `src/shared/config/`:

### table.config.ts
- Заголовки таблицы
- Опции пагинации

### form.config.ts
- Конфигурация полей формы
- Сообщения валидации
- Сообщения успеха/ошибки

## Запуск проекта

### Установка зависимостей

```bash
npm install
```

### Запуск dev сервера

```bash
npm run dev
```

Приложение доступно: http://localhost:3000
